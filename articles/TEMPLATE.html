Article Template

Summary: Briefly summarize the project in one sentence.

Step 1. Compose the Content

The article needs...

  • Header image, 1280px wide PNG (any height)
  • Title
  • Summary
  • Description (for the homepage)
  • Thumbnail, 200px × 200px PNG (for the homepage)
  • Captions for every image
  • Spell check

Step 2. Format the Article for Speed-Reading

Use the mark element to highlight the sentences you definitely want the reader to read.

Take a pass through the article and write it to have a coherent narrative if the user were to only read...

  • Headers
  • Image captions
  • Highlighted text
  • Bold text

Step 3. Convert the Article into HTML

Copy this HTML file into the root directory and rename it with the article's title using kebob case.

  • Use section tags
  • Every section should have a header
  • Update the <title>, og:title, and twitter:title using the format: "<Article Title> | Edwin Choate UX"
  • Images should go in figure elements, with figcaptions
  • All img elements should have alt text
  • All external links should open in new tabs (target="_blank")
  • Use em and strong in the prose
  • Use relative links for internal links (href and src)

Step 4. Update the Sitemap

  • Update sitemap.xml using xml-sitemaps.com
  • Update the Read More section of every article on the site
    • The order of the projects in the Read More section should match the order of the projects on the homepage
    • The link to the article you just finished reading should be disabled on each article

Step 5. Test

  • Manually check links
  • Reader mode
  • Dark mode
  • Screen sizes down to 320px
  • Print view
  • W3C Markup Validation Service
  • W3C CSS Validation Service
  • W3C Link Checker
  • Web Accessibility Evaluation Tool (WAVE)
